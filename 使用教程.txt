# 局域网主控与被控工具使用教程

适用版本：`2026.07.20-21` 及以上。

本工具由两个程序组成：

- `主控端.exe`：在管理电脑上运行，用于搜索被控端、选择电脑、打开程序以及提交 Riot 登录账号。
- `被控端Agent.exe`：在每台被控制电脑上运行，监听主控端发送的请求。

## 一、使用前准备

### 1. 网络要求

主控电脑和被控电脑必须能够通过局域网互相访问。

- 实体电脑通常连接到同一个路由器或交换机。
- VMware 虚拟机建议使用桥接模式；使用 NAT 模式时，主控端必须扫描对应的 VMware NAT 网段。
- 被控端默认监听 TCP 端口 `8765`。
- 主控端和所有被控端的 Token 必须完全一致。

### 2. 文件要求

把 `release` 文件夹完整复制到主控电脑和每台被控电脑，不要只复制单个 `.exe`。至少需要保留：

```text
主控端.exe
被控端Agent.exe
agent_config.json
hosts.txt
scripts\
```

### 3. 设置 Token

使用文本编辑器打开每台电脑上的 `agent_config.json`，修改：

```json
"token": "请换成一段较长的随机字符串"
```

主控电脑和所有被控电脑必须填写相同的 Token。修改配置后需要重启相关程序。

## 二、启动被控端

在每台被控电脑上执行以下操作：

1. 运行 `被控端Agent.exe`。
2. 确认窗口标题版本与主控端版本一致。
3. 监听地址保持 `0.0.0.0`。
4. 端口保持 `8765`。
5. 点击“启动 Agent”。
6. 状态显示“运行中：0.0.0.0:8765”即表示启动成功。

同一台电脑只能运行一个 Agent。如果出现 `WinError 10048`，请在任务管理器中结束旧版 Agent 或以前通过 `python agent.py` 启动的进程，然后重新启动。

### 放行 Windows 防火墙

管理员 PowerShell：

```powershell
New-NetFirewallRule -DisplayName "LAN Agent 8765" -Direction Inbound -Protocol TCP -LocalPort 8765 -Action Allow
```

没有 PowerShell 时，可在管理员 CMD 中执行：

```bat
netsh advfirewall firewall add rule name="LAN Agent 8765" dir=in action=allow protocol=TCP localport=8765
```

### 检查被控端状态

在主控电脑浏览器中访问：

```text
http://被控电脑IP:8765/health
```

当前版本应返回类似内容：

```json
{"ok": true, "agent": "LanScriptAgent/1.5"}
```

## 三、启动主控端

1. 运行主控电脑上的 `主控端.exe`。
2. 确认窗口标题为当前发布版本。
3. Token 通常会从同目录的 `agent_config.json` 自动加载。
4. “文件名称”默认填写为 `Riot Client`。

## 四、自动搜索被控端

1. 在“扫描网段”中填写要搜索的网段，例如：

   ```text
   192.168.1.0/24
   ```

2. 端口填写 `8765`。
3. 超时秒数可保持 `0.6`。
4. 点击“自动检索”。
5. 扫描到的设备会显示在右侧输出区域，例如：

   ```text
   [发现] 192.168.1.20:8765 LanScriptAgent/1.5（待添加）
   ```

6. 点击“添加扫描结果”，扫描结果才会进入主机列表。

自动扫描一次只扫描输入框中的一个网段。电脑存在多个网卡时，需要分别扫描对应网段，例如：

```text
192.168.1.0/24
192.168.61.0/24
192.168.187.0/24
```

也可以在“添加被控主机”中手动填写：

```text
192.168.1.20:8765
```

## 五、为每个 IP 设置 Riot 账号

账号和密码由主控端手动填写，每个 IP 可以使用不同账号。

1. 在主机列表中单击一台主机所在行。
2. 在“当前主机 Riot 登录账号”区域填写账号和密码。
3. 点击“应用”。
4. 表格中会显示账号，密码列显示“已填写”。
5. 对其他 IP 重复以上步骤。

密码不会写入 `hosts.txt`，只保存在主控端当前运行的内存中。关闭或重启主控端后，需要重新填写密码。

“选择”列控制该主机是否参与批量操作。可使用“全选”“反选”“删除”“清空”和“保存”管理主机列表。

## 六、启动并登录 Riot Client

1. 确认每台被控电脑已经启动 Agent。
2. 确认被控电脑桌面处于已登录且未锁定状态。
3. 在主控端勾选要操作的主机。
4. 确认每台被勾选主机都已应用账号和密码。
5. 点击“启动并登录 Riot”。

被控端将按以下顺序执行：

1. 查找 `RiotClientServices.exe`；找不到窗口时自动启动 Riot Client。
2. 捕获标题为 `Riot Client`、类名为 `Chrome_WidgetWin_1` 的窗口句柄。
3. 等待同一个窗口句柄连续稳定 5 秒；句柄变化时重新计时。
4. 将 Riot 窗口置于前台。
5. 单击“使用者名称”输入框并输入账号。
6. 按一次 `Tab` 切换到密码框并输入密码。
7. 按 `Enter` 提交登录。

如果 Riot 服务残留但窗口已经退出，被控端会自动重启 Riot Client 并重新捕获窗口，最多尝试 3 次。

登录输入期间不要操作被控电脑的键盘和鼠标，也不要锁定桌面。验证码、二次验证和账号异常需要人工处理，程序不会绕过这些验证。

## 七、普通文件或程序打开功能

除 Riot 登录外，也可以让被控电脑搜索并打开其他文件或程序：

1. 在“文件名称”中填写文件名或程序名，不填写完整路径。
2. 勾选要操作的主机。
3. 点击“逐台搜索并打开”。

例如：

```text
README.md
Riot Client
倚天2：觉醒
```

搜索范围由被控电脑的 `agent_config.json` 中 `file_search_roots` 控制。

## 八、常见问题

### 扫描不到被控端

依次检查：

1. 被控端是否已经点击“启动 Agent”。
2. IP 和扫描网段是否正确。
3. 主控端和被控端是否能够互相访问。
4. 防火墙是否放行 TCP `8765`。
5. 虚拟机是否使用正确的桥接或 NAT 网段。

### `HTTP=401 unauthorized`

表示 Token 不一致，或者主控端与被控端版本不匹配。

- 确保两端 `agent_config.json` 中的 Token 完全一致。
- 确保主控端和被控端使用同一发布版本。
- 访问 `/health` 检查 Agent 版本。
- 结束旧版 Agent 和旧的 `python agent.py` 进程。

### `WinError 10048`

端口 `8765` 已被其他 Agent 占用。关闭旧进程后重新启动，不要同时运行多个 Agent。

### `unknown encoding: idna`

表示使用了缺少 IDNA 编码的旧主控端。请升级到 `2026.07.20-21` 或更高版本，并确认窗口标题确实是新版。

### `riot_window_not_found`

被控端未找到 Riot 登录窗口。

- 确认桌面未锁定。
- 检查 Riot Client 是否正常安装。
- 必要时在 `agent_config.json` 的 `riot_client_executable` 中填写完整路径。

示例：

```json
"riot_client_executable": "D:\\Riot Games\\Riot Client\\RiotClientServices.exe"
```

### `cannot_focus_riot_username` 或 `cannot_focus_riot_password`

Riot 窗口没有获得输入焦点。执行登录时不要操作被控电脑，关闭可能抢占前台的弹窗后重试。

### 找到文件但没有打开

检查被控端返回的目标路径、快捷方式目标和启动状态。游戏启动器还可能要求管理员权限、运行库或已登录的交互桌面。

## 九、升级程序

升级时建议按以下顺序操作：

1. 关闭旧版主控端和被控端。
2. 记录原 `agent_config.json` 中的 Token 和自定义搜索路径。
3. 替换 `主控端.exe` 和 `被控端Agent.exe`。
4. 使用新版 `agent_config.json`，并恢复相同 Token 和需要保留的搜索路径。
5. 先启动所有被控端，再启动主控端。
6. 确认两端窗口标题版本一致。

主控端升级后，所有被控端也应同步升级，避免鉴权协议和功能不一致。

## 十、安全注意事项

- 不要继续使用默认 Token。
- 不要把 Token、账号和密码发给无关人员。
- Riot 密码只保存在主控端内存中，不会写入主机列表或运行日志。
- 登录请求使用 AES-GCM 加密和 HMAC 鉴权。
- 只在可信局域网中使用，并通过防火墙限制 `8765` 的访问范围。
